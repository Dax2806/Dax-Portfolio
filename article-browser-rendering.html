Inner Workings of Browser - Rendering 101

Published June 16, 2025

When an HTML file is created and CSS is embedded, we can directly open it to the browser, to view the styled webpage. But how does the web browser actually process and renders the file? This article explores the inner working of web browsers - from loading files to final display. Before exploring the process lets get the introduction with web browser and its architecture, and the Document Object Model (DOM).

What is a Web Browser?

Web browser is a software which is responsible for loading web files and displaying information in a user-friendly interface.

The web browser connects the user and the server. It's a highly advanced software which has the ability to display huge potential information with a simple user-friendly interface. It is an intermediate part of a web which connects the HTML, CSS, JS files and other resources with the server. It is an interaction tool for using the internet.

Examples of web browsers are Chrome, Safari, Firefox, Edge, Brave, etc.

A Web Browser is divided into several components: User Interface; Data Storage; Browser Engine - Rendering Engine, JavaScript Engine, Timers, Networking Layer.

Browser Architecture

Browser architecture is designed to provide a faster and secure platform that helps users to interact with the internet. The Browser is divided into several parts:

User Interface - UI or the user interface allows all kinds of personalization to the browser, which can be initiated by the users according to the usage. Every individual has different interests and so different kinds of browser setup. The UI displays the basic features like bookmarks, groups, extensions, themes, URLs, etc.

Data Storage - A large part of the browser goes into storing different types of data such as user preferences, browsing history, passwords, activity controls, and other regular user data.

The Browser Engine - It keeps a note on which button is clicked by the user, which URL is asked to parse, and how the web content will be processed and displayed on the browser.

The Rendering Engine - It interprets and renders the web content. The browser isn't capable of read the HTML files directly, so the rendering engine helps the browser to convert the HTML tags (i.e. <p>, <body>, <h1>, <nav>, etc) into the nodes.

The Networking Layer - The Networking layer deals with the communication part. When the user clicks on a URL, the network layer initiates the HTTP request and loads the webpage. While surfing over the internet, a cookie notification pops up in the window. The Networking is behind the scenes with the cookies.

JavaScript Engine - The HTML and CSS files just structure and add styles to the webpage. Javascript is responsible for adding interactivity in it. Therefore, JS Engine balances the dynamic behaviour in web pages.

Document Object Model (DOM) & CSS Object Model (CSSOM)

The DOM is the programming interface for the web documents. It represents the structure of a webpage as a tree-like model where each tag (i.e. <p>, <body>, <h1>, <nav>, etc) is a node in a tree. The DOM allows JavaScript to interact with HTML and CSS files. Job of a browser is to display the data and allow interactivity.

How does the browser load the HTML in the form of NodeList?

The Rendering engine comes in the picture. With the help of the rendering engine the browser loads the Nodelist.

If your HTML looks like this:

<!DOCTYPE html>
<html>
  <head>
    <title>Page Title</title>
  </head>
  <body>
    <h1>Hello World</h1>
    <p>This is a paragraph.</p>
  </body>
</html>

How does DOM work:

  1. Browser loads HTML files.
  2. After accessing the structure (HTML) and styling (CSS), the rendering engine converts the tags and elements into raw bytes, i.e. The binary data.
  3. Tokenization - The HTML elements are treated as tokens.
  4. These tokens are then bifurcated to different objects, and every element is structured according to hierarchy. This conversion of objects into structure is known as browser model.
  5. The models are further connected according to the relation and hierarchy.
  6. Hence, The Nodelist is ready.
  7. The DOM of HTML is ready.

After completing the rendering process of HTML, the engine starts the rendering of CSS, and it is known as CSSOM. The browser loads all the linked CSS Styling. It creates the CSSOM tree.

The Rendering process of HTML and CSS still doesn't help us to display the webpage in the browser. Then, The DOM and CSSOM combine together to form the Render tree. The Render tree performs certain calculations to fulfil the CSS requirement of a webpage.

Running JavaScript to DOM & CSSOM

The DOM and CSSOM files are ready. Thus, JS files are runnable for adding interactivity to structure. JavaScript is usually added after the content of HTML and CSS file, as the JS files have capability to manipulate the DOM & CSSOM. So, usually the HTML and CSS files are rendered before connecting the JavaScript files.

If we talk about JS & DOM, JavaScript has the higher priority, and if we talk about JS & CSSOM, CSSOM has the higher priority. Thus, CSS can be sometimes a bottleneck while giving interactions.

Summary:

  • URL is entered
  • HTTP/HTTPS request is sent
  • Browser starts rendering
  • DOM Tree
  • CSSOM Tree
  • Render Tree (DOM + CSSOM)
  • JavaScript execution
