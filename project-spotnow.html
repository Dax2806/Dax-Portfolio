SpotNow: City Discovery Platform

Tech Stack: Node.js / Express / MongoDB / HTML+CSS+JS / Leaflet.js

Project Overview

SpotNow is a modern, community-driven city discovery platform designed to help users find and share "hidden gems"—unique, lesser-known locations such as tucked-away cafés, secret gardens, rooftop views, and street art. The project connects local explorers with travelers by providing an interactive, map-based interface for authentic urban exploration.

Key Features

  • Interactive Exploration: Users can browse a filterable grid of spots (Cafés, Parks, Viewpoints, Art) accompanied by real-time map markers powered by Leaflet.js.
  • Geographical Pinning: A dedicated "Add Spot" interface allows users to click directly on a map to capture precise latitude and longitude coordinates.
  • Community Contributions: Registered users can submit new spots by providing a name, location description, category, and a photo upload.
  • Secure Authentication: Features a complete Signup and Login system with hashed passwords to protect user identities.
  • Privacy & Access Control: Implements a global "Auth Guard" (Bouncer Script) that redirects unauthorized visitors to the login page, ensuring the platform's content is exclusive to the community.
  • Dynamic UI State: The navigation bar automatically updates to display the logged-in username and a logout option once a user is authenticated.

Technical Architecture

SpotNow is built on a robust full-stack architecture, separating client-side presentation from server-side data management.

Layer Technology Used Purpose
Frontend HTML5, CSS3, JavaScript (ES6+) Structural layout and interactive client-side logic.
Styling Bootstrap 5 & Custom CSS Responsive design and "Glassmorphism" aesthetic.
Mapping Leaflet.js & OpenStreetMap Interactive map rendering and coordinate selection.
Backend Node.js & Express RESTful API routing and server-side logic.
Database MongoDB Atlas & Mongoose Cloud NoSQL storage for Users and Spots.
Auth Bcryptjs Secure one-way password hashing for the database.
Storage Multer Middleware for handling multipart/form-data (image uploads).

Component Breakdown

1. Frontend Experience

  • Visual Identity: The project uses the DM Sans font for a clean look and a warm primary gradient (#ff7645 to #ff9a6c) throughout the UI.
  • Glassmorphism Design: High-end visual effects are achieved using semi-transparent backgrounds (rgba(255, 255, 255, 0.92)), backdrop filters, and soft shadows to create a layered "glass" feel for cards and forms.
  • Responsive Layouts: The application is built to be mobile-friendly, utilizing Bootstrap's grid system to stack content effectively on smaller screens.

2. Backend Infrastructure

  • Data Models: Mongoose schemas define the structure for Spots (storing name, location, category, coordinates, and image URLs) and Users (storing unique usernames and hashed passwords).
  • Environment Security: Sensitive data, such as the MongoDB connection string (MONGO_URI), is managed through Environment Variables to prevent exposure in the source code.
  • File Handling: Images uploaded by users are processed by Multer and served statically via an /uploads route on the backend server.

3. Security Measures

  • Password Hashing: Passwords are never stored in plain text; instead, they are processed with 10 salt rounds using Bcryptjs.
  • Client-Side Session: User state is maintained in localStorage, allowing the application to persist the login status even if the browser is refreshed.
  • Protected Routing: Every internal page (Index, Explore, Add-Spot, About) contains a <head>-level script that verifies the presence of a user token before rendering content.

Deployment Workflow

The application follows a modern cloud deployment strategy:

  • Frontend: Hosted on Netlify, providing fast delivery of the static HTML/CSS/JS assets.
  • Backend: Hosted on Render as a web service, handling the live API requests and database communication.
  • Database: Managed via MongoDB Atlas, with network access configured to allow connections from the Render server IP environment.
