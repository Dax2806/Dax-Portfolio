How Secure is The Web? HTTP vs HTTPS vs SSH

Published August 12, 2025

What is Internet Protocol?

Internet protocols are standardized rules that enable computers and other devices to communicate across the Internet. They break information into packets, address them, and route them to the correct destination. These protocols ensure that data is transmitted, received, and interpreted accurately between different systems.

There are various types of Internet protocols, each designed for a specific function—such as transferring files, sending emails, or securing communication.

Every device connected to the Internet has a unique IP address that identifies the sender and receiver, making reliable communication possible.

What is IP Addressing?

An IP (Internet Protocol) address is a unique numerical identifier assigned to every device on a network. It allows the Internet to distinguish between different computers, routers, and websites, ensuring data reaches the correct destination. IP also defines the rules for how data is structured, addressed, and transmitted across networks.

There are two versions of IP addresses:

  • IPv4, which is 32 bits long.
  • IPv6, which is 128 bits long and created to overcome IPv4 address exhaustion.

IPv4 addresses are divided into five classes:

  • Class A: For very large networks
  • Class B: For medium-sized networks
  • Class C: For small networks
  • Class D: Reserved for multicasting (sending a packet to multiple devices at once)
  • Class E: Reserved for experimental and future use

IP addressing works much like a home address system—without a unique address, devices cannot send or receive information. Every IP address performs two key functions: Identification (identifying the specific device) and Location addressing (specifying where the device is located in the network).

Need for an Internet Protocol

When data is exchanged over the Internet, the sender and receiver may be on different networks with different data transfer speeds. To ensure smooth communication, protocols manage flow control and access control.

For example, if sender X transmits data at 10 Mbps while receiver Y can only accept 5 Mbps, data loss will occur. To prevent this, the receiver notifies the sender about the speed difference, allowing the sender to reduce its transmission rate. This coordination ensures reliable data delivery.

Access control determines which device can use a shared communication link at a given moment. Without it, multiple devices sending data simultaneously could cause collisions, leading to corrupted or lost data.

Internet protocols are essential because they:

  • Uniquely identify devices
  • Ensure accurate data delivery
  • Route data efficiently across networks
  • Enable communication between different systems

Working of Internet Protocol

When data is sent over the Internet, the Internet Protocol (IP) ensures that it reaches the correct destination efficiently and accurately. It works through several steps:

  1. Dividing Data into Packets: Large data is broken into small units called packets. Each packet contains part of the information along with the destination address.
  2. Addressing: Every device has a unique IP address that identifies where the data is coming from and where it should go.
  3. Routing Packets: Packets travel through multiple routers, which guide them toward the correct destination—similar to how mail is sorted through various post offices.
  4. Reassembling Data: Once all packets reach the destination, they are combined to reconstruct the original message or file.
  5. Handling Missing Packets: If any packets are lost during transmission, the system requests them again to ensure complete data delivery.

A good way to understand IP is to imagine sending a letter. The sender and receiver addresses on an envelope help the postal service deliver it correctly. Similarly, devices use IP addresses—like 192.168.1.1—as their “home addresses.”

Example: When you open a website such as www.google.com:

  1. Your computer asks a DNS server for Google’s IP address.
  2. It then sends a request to that IP.
  3. Google’s server processes the request and sends data back.
  4. Your browser displays the website.

Different Types of Internet Protocols

  • TCP / IP = Transmission Control Protocol
  • HTTPS = Hypertext Transfer Protocol Secure
  • HTTP = Hypertext Transfer Protocol
  • SMTP = Simple Mail Transfer Protocol
  • PPP = Point-to-Point Protocol
  • FTP = File Transfer Protocol
  • SFTP = Secure File Transfer Protocol
  • SSH = Secure Shell

HTTP (Hypertext Transfer Protocol)

HTTP (HyperText Transfer Protocol) is the primary protocol used to transfer hypertext and other web content across the Internet. Defined by the World Wide Web (WWW), it specifies how information should be formatted, transmitted, and how web browsers and servers should respond to various requests.

Whenever you browse the web, you are indirectly using HTTP to load text, images, videos, and other multimedia resources.

How HTTP Works (Basic Flow):

  1. The client (browser) sends a request to the server asking for a webpage or resource.
  2. The server receives the request and locates the required data.
  3. The server sends back a response, which includes the content and a status code.
  4. The browser processes this response and displays the webpage to the user.

Is HTTP secure? No. HTTP transfers data in plain text, which makes it vulnerable to interception. To ensure security, encryption is used in HTTPS (HTTP Secure), the safer version of the protocol.

HTTPS (Hypertext Transfer Protocol Secure)

HTTPS (HyperText Transfer Protocol Secure) is the secure version of HTTP. It uses SSL/TLS encryption to protect data exchanged between a user’s browser and a website, ensuring privacy, authenticity, and data integrity. Websites that handle sensitive information—such as credit card numbers, passwords, or OTPs—must use HTTPS to prevent unauthorized access or data theft.

What Makes HTTPS Secure?

HTTPS relies on SSL/TLS to:

  • Encrypt all data sent and received
  • Verify the website’s identity through digital certificates
  • Prevent tampering, spying, or data interception during transmission

How HTTPS Works:

  1. The browser requests a secure connection to the website.
  2. The website sends its SSL certificate, containing its identity and public key.
  3. The browser checks whether the certificate is valid and trusted.
  4. Once verified, an encrypted communication channel is established.
  5. All data is then exchanged securely.

To enable HTTPS, a website needs an SSL certificate issued by a Certificate Authority (CA). Before entering sensitive information online, users should always check that the website URL begins with HTTPS, indicating a secure connection.

Secure Shell (SSH)

SSH (Secure Shell) is a secure network protocol used for remote login and managing devices over the internet. It provides an encrypted connection that protects all data exchanged between a user and a remote system, making it much safer than older protocols like Telnet.

SSH uses public-key cryptography for authentication and encryption, ensuring that only authorized users can access the remote device. It also supports secure file transfers through SCP (Secure Copy) and SFTP (Secure File Transfer Protocol).

SSH is widely used on Unix/Linux systems and is also available on Windows. System administrators, developers, and other technical users commonly rely on SSH to manage servers and network devices remotely.

How SSH Works:

  1. The user runs an SSH command to connect to a remote system.
  2. Both systems exchange public/private keys for authentication.
  3. Once verified, an encrypted session is established.
  4. The user can then execute commands or transfer files securely.

Introduction to SSL & TLS

SSL (Secure Sockets Layer) is a cryptographic protocol designed to secure communication over the internet by encrypting data, authenticating identities, and ensuring data integrity. Although SSL was widely used in the past, it has been replaced by TLS (Transport Layer Security), which offers stronger security and improved performance. Today, most references to “SSL” in practice actually mean SSL/TLS.

What SSL/TLS Do:

  • Encrypt Data: All transmitted information is scrambled, so intercepted data appears unreadable.
  • Authenticate Identity: Through a process called the SSL/TLS handshake, devices exchange certificates to confirm they are communicating with the correct party.
  • Ensure Data Integrity: Digital signatures ensure that data has not been altered during transmission.

How SSL/TLS Work:

  1. The browser connects to a website using https://.
  2. The website sends its SSL/TLS certificate, issued by a trusted Certificate Authority (CA).
  3. The browser verifies the certificate’s authenticity.
  4. After verification, both sides generate encryption keys and establish a secure session.
  5. Encrypted data is then safely exchanged between the browser and the server.

Conclusion

Internet protocols play a vital role in ensuring seamless, secure, and efficient communication across global networks. They define how data is addressed, transmitted, routed, and delivered, enabling devices—from computers to servers—to interact reliably. Protocols like IP, TCP/IP, HTTP, and DNS form the foundation of basic communication, while secure protocols such as HTTPS, SSH, and SSL/TLS protect data integrity and privacy in an increasingly interconnected world.

Together, these protocols create a structured ecosystem that supports everything from browsing websites to remote server management and secure transactions. Understanding how they work helps us appreciate the complexity of the internet and highlights the importance of using secure, modern protocols to keep online communication safe and trustworthy.
